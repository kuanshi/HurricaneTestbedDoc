2.2.1. Inputs¶

The following files must be provided by the user to execute the workflow.

2.2.1.1. Configuration File¶

JSON file containing simulation details and specifies the application selected for each workflow step.

properties

  • runDir

Directory where input_data folder is located.

type

path

  • localAppDir

Directory where applications folder is located.

type

path

  • units

Dictionary of strings which specifiy units for force, length, and time.

type

string

default

N, m, sec

  • outputs

Indicator (True/False) of which workflow outputs (EDP, DM, DV) are provided in the simulation.

type

bool

default

true

  • Applications

Dictionary of applications selected for each workflow step, with their associated application-specific inputs.

type

dict

2.2.1.2. Building Source File¶

CSV file containing building properties for the response simulation and damage/loss estimation steps of the workflow. The following headers must be included (with any one of the header options listed):

properties

  • id

ID number of the building.

type

int

  • ‘Longitude’, ‘longitude’, ‘lon’, ‘Lon’

Longitude coordinate of the building site.

type

float

  • ‘Latitude’, ‘latitude’, ‘lat’, ‘Lat’

Latitude coordinate of the building site.

type

float

  • ‘Stories’, ‘stories’

Number of stories in the building.

type

int

  • ‘Year Built’, ‘yearbuilt’, ‘yearBuilt’

Year that the building was built.

type

int

  • ‘Structure Type’, ‘structure’, ‘structureType’

Structural type of the building.

type

string

  • ‘Type ID’, ‘occupancy’, ‘occupancyType’

HAZUS occupancy class of the building.

type

string

  • ‘Area’, ‘areafootprint’, ‘areaFootprint’, ‘area’

Floor plan/footprint area of the building.

type

float

  • ‘Replacement Cost’, ‘replacementCost’

Cost of replacement of the building. If provided as ‘1.0’, then the DL assessment will return an estimate losses as a percentage of the total replacement cost.

type

float

  • model_params

(Optional) Columns for additional structural parameters used in the build_model function for creating a structural analysis model of the building.

type

float

2.2.1.3. Model File¶

If the simulation application chosen is OpenSees-Simulation or OpenSeesPy-Simulation, then a model file must be provided. It consists of an OpenSees/OpenSeesPy script with functions for building and analyzing a custom structural model. See the Simulation Applications page for more details on application-specific inputs.

2.2.1.4. EDP specifications file¶

JSON file specifying types, locations, and directions of EDPs to record in the structural model during response simulation.The EDP specifications file is only necessary as an input if UserDefinedEDP is chosen as the EDP application.

2.2.1.5. Event Files¶

To specify the type, location, and intensity of each event, the following files must be provided in the input_data/records folder.

properties

  • EventGrid

CSV file which specifies the coordinate (latitude/longitude) location of each event site.

properties

  • sta

Name of event file associated with the given event site. If running time history analysis, then the name of the site file is provided instead.

type

string

  • lon

Longitude coordinate of the site location.

type

float

  • lat

Latitude coordinate of the site location.

type

float

  • Event IM Files

CSV or JSON files containing information on the intensity measure of the event.

properties

  • name

Name of ground motion record.

type

string

  • dT

Time step of ground motion recording.

type

float

  • data_x

List of acceleration points in the x-direction ground motion record.

type

list

  • data_y

List of acceleration points in the y-direction ground motion record.

type

list

  • PGA_x

Value of peak ground acceleration in the x-direction ground motion record.

type

float

  • PGA_y

Value of peak ground acceleration in the y-direction ground motion record.

type

float

  • Site Files

CSV files listing the names of the event file(s) assigned to the site for time history analysis. This file can also provide other properties, such as a ground motion scale factor.

properties

  • GM_file

Name of the ground motion event file (for Earthquake hazard events).

type

string

  • factor

Factor for scaling the ground motion.

type

float

2.2.1.6. Hazard Simulation¶

This is an optional pre-processing application for users to simulate regional earthquake events and prepare the Event Files in the previous section. Users need to provide the a configuration json file following the schema summarized below.

properties

  • Site

Location information about the sites of interest.

properties

  • Type

Information source type, e.g., ‘From_CSV’

type

string

  • input_file

Filename of the site information file, e.g., ‘SiteFile.csv’.

type

path

  • min_ID

The first site ID in the site information file.

type

int

  • max_ID

The last site ID in the site information file.

type

int

  • Scenario

Definitions of the hazard scenario(s) and earthquake source(s).

properties

  • Type

Hazard type which in this case is ‘Earthquake’.

type

string

  • Number

Number of scenarios in the simulation.

type

int

  • Generator

Method of generating the scenario, currently supported: ‘Selection’.

type

string

  • EqRupture

Earthquake rupture selection criteria

properties

  • Type

Data source, currently supported: ‘ERF’

type

string

  • Model

Earthquake source model type, e.g., ‘WGCEP (2007) UCERF2 - Single Branch’.

type

string

  • Name

Keywords in the earthquake name, e.g., ‘San Andreas’.

type

string

  • min_Mag

Minimum magnitude of interest.

type

float

  • max_Mag

Maximum magnitude of interest.

type

float

  • max_Dist

Maximum distance (to the geometric centroid of the sites) of interest.

type

float

  • Event

Specifying the ground motion prediciton models and intensity measures for selecting ground motion records.

properties

  • NumberPerSite

Number of simulated ground motion records per site.

type

int

  • GMPE

Defining the ground motion prediction model.

properties

  • Type

Ground motion prediction model name, e.g, ‘Chiou & Youngs (2014)’.

type

string

  • Parameters

Optional configurations for specific ground motion prediction models.

type

list

  • CorrelationModel

Defining the intensity measure correlation model.

properties

  • SaInterEvent

Inter-event correlation model name, e.g, ‘Baker & Jayaram (2008)’.

type

string

  • SaIntraEvent

Intra-event correlation model name, e.g, ‘Markhvida et al. (2017)’.

type

string

  • IntensityMeasure

Defining the intensity measure type and periods (if any).

properties

  • Type

Intensity measure name, e.g, ‘SA’.

type

string

  • Periods

A float number list of periods for Sa.

type

list

  • ScalingFactor

Scaling factor limits in ground motion selection.

properties

  • Maximum

Maximal allowable scaling factor.

type

float

  • Periods

Minimal allowable scaling factor.

type

float

  • SaveIM

Choice to save simulated regional intensity measures.

type

boolean

  • Database

Ground motion database name, currently supported: ‘NGAWest2’.

type

string

  • UserName

User name on PEER NGA-West database (optional for downloading records).

type

string

  • UserPassword

User password on PEER NGA-West database (optional for downloading records).

type

string

  • OutputFormat

Output data format, currently supported: ‘SimCenterEvent’ (optional for downloading records).

type

string

  • Directory

Specifying the directories for the application.

properties

  • Work

The main directory of the Hazard Simulation application.

type

path

  • Input

The directory of the site information file.

type

path

  • Output

The diectory of the saved records.

type

path
